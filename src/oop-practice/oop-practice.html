1 Prototype based Easy

OOP using prototypes on plain objects. Try changing the names or adding a method, then run it.

💡 Hint

Object.setPrototypeOf(child, parent) makes parent's properties available on child via the prototype chain.

2 Inheritance (extends) Medium

A LibraryBook class that extends Book, adds an isBorrowed flag with borrow() / returnBook(), and overrides getInfo().

💡 Hint

Call super(title, author, year) in the constructor. Use super.getInfo() inside the overridden method.

3 Getters & Setters Medium

A Rectangle with getters for area / perimeter and a width setter that rejects non-positive values.

💡 Hint

Use _width / _height as backing fields. get area() { }, set width(v) { if (v > 0) … }

4 Static Methods Medium

A TemperatureConverter with a static FACTOR = 9/5 and static methods celsiusToFahrenheit(c) / fahrenheitToCelsius(f).

💡 Hint

static methodName() { … } — call on the class, not on an instance.

5 Private Fields (#) Hard

A BankAccount with a private #balance field, deposit(amount), withdraw(amount) (must not go below 0), and a balance getter.

💡 Hint

Declare #balance at the top of the class body. Access it with this.#balance inside methods.

6 Polymorphism Hard

A base Shape with getArea(), extended by Circle (π·r²) and Square (s²), each overriding getArea().

💡 Hint

Use Math.PI for pi. Call super() in subclass constructors. Polymorphism lets you call the same method on different shapes.
